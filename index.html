Inicio

A continuación se documenta los requisitos necesarios para desplegar el proyecto en un entorno local.

El primer paso será realizar la instalación de un servidor apache que proporciona los componentes esenciales y necesarios como: PHP, MariaDB y phpMyAdmin entre otros. Recomendamos realizar la instalación de Xampp, para hacerlo puedes acceder a su sitio web para obtener la versión 8.2.4.

Finalizada la instalación de Xampp, iniciaremos el servicio de Apache y MySQL desde el panel de control de Xampp:

El siguiente paso será crear una base de datos. Para ello accederemos desde el navegador al siguiente enlace: http://localhost/phpmyadmin/ y presionamos sobre el botón "Nueva".

La nombraremos "esuru" utilizando la configuración utf8mb4_general_ci como se muestra en la siguiente imagen:

Utilizamos Composer para el manejo de dependencias del proyecto. Si se encuentra en un entorno de Windows puedes descargar el instalador aquí. Si te encuentras en un ambiente Linux o Mac, puedes acceder a su documentación oficial.

Una vez finalizada la instalación de composer y teniendo el servidor apache encendido podemos continuar.

Despliegue

Clone el repositorio en su directorio actual y descomprima el archivo: 

$ git clone https://github.com/German-Cinelli/ta3-iti-2023

Nota

Si no tiene git instalado, puedes descargar el proyecto de forma manual desde https://github.com/German-Cinelli/ta3-iti-2023

Dentro de la raiz del proyecto cambie el nombre del archivo .env.example a .env. Posteriormente abralo con cualquier editor de textos y modifique los valores de las siguientes constantes para que queden de la siguiente manera:

  • DB_DATABASE=esuru
  • DB_USERNAME=root
  • DB_PASSWORD=

Navegue hasta el directorio raiz del proyecto con el comando cd y ejecute: 

$ composer install

Genere una clave de cifrado única para la aplicación: 

$ php artisan key:generate

Crea un enlace simbólico para el almacenamiento de archivos: 

$ php artisan storage:link

Genere toda la estructura de bases de datos. El parámetro --seed inserta los datos de prueba necesarios. 

$ php artisan migrate --seed

Para levantar el proyecto ejecute: 

$ php artisan serve

Abra en su navegador el enlace proporcionado en su interfaz de comandos: http://127.0.0.1:8000/

Nota

Puedes detener el servidor presionando ctrl+c en su terminal.

Roles

Los roles permiten asignar diferentes niveles de autoridad y permisos a los usuarios con el fin de controlar el acceso y la funcionalidad dentro del sistema. Determinan qué acciones y recursos pueden realizar los usuarios en funcion al rol.

Jerarquía

  • Administrador: Acceso a todas las funcionalidades del sistema. Puede ingresar usuarios, servicios, listar proveedores, clientes, etc.
  • Soporte: Encargados de la resepción y revisión de reportes realizados por los clientes. Puede acceder a los datos personales de cada proveedor junto al listado e información de cada trabajo.
  • Prestador: Los prestadores de servicios pueden acceder a su perfil para editar sus datos peronales, formación académica y experiencia laboral. Además detallan que tipo de servicios brindan y acceden a los datos de contacto del cliente cuando les realizan una solicitud de tarbajo por sus servicios.
  • Cliente: Se registran en la plataforma por cuenta propia. Pueden listar servicios y acceder al perfil de los prestadores.

Permisos

Se utilizan para controlar el acceso y las acciones que los usuarios pueden realizar dentro de la plataforma dependiendo del rol. Permiten mostrar u ocultar elementos de la interfaz gráfica para proteger funcionalidades específicas y realizar validaciones para determinar si un usuario tiene acceso o no a un recurso.

Servicios

En nuestra plataforma de conexión entre clientes y especialistas, adoptamos la estrategia de predefinir los servicios ofrecidos, brindando a los prestadores la libertad de elegir y personalizar cuáles de ellos desean brindar.

Una ventaja importante es que agiliza las búsquedas y el filtrado, lo que ahorra tiempo y esfuerzo al evitar la necesidad de revisar perfiles individualmente para verificar si un prestador ofrece el servicio requerido.

Además, al permitir que los prestadores seleccionen los servicios que desean brindar, se logra una mayor precisión en las ofertas. Esto garantiza que los profesionales estén proporcionando exactamente lo que desean ofrecer, evitando cualquier malentendido con los clientes y asegurando una claridad mutua sobre el servicio en cuestión.
